This template demonstrates the Token-Diet v3.0 prompt assembly structure:

*   `[SYS — cached]`: For static system instructions that can be cached.
*   `Context`: Placeholder for pruned retrieval results (max 1200 tokens).
*   `State`: Structured JSON for dynamic facts and constraints, replacing verbose history.
*   `Task`: User input, linguistically compressed (e.g., "Analyze data. Summarize concise.").

This structure ensures optimal token usage by prioritizing:
1.  **Caching** static elements.
2.  **Pruning** irrelevant context.
3.  **Structuring** dynamic state.
4.  **Compressing** only the essential task instruction.
